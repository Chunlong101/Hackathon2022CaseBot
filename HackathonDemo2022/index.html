Case Assignment Bot

Case assignment and workload balance could be a challenge work in a team which supports multiple products and has people with multiple specialties. It's difficult to consider all the situations like support engineers who are on shift, planned leave, urgent leave, who is in a hot task, training, who is good at what technologies. The challenge is even bigger when all members are working from home. Besides, as a manager it is difficult to understand the real workload for each team member when do performance review. The intention for this solution is to leverage multiple modern tools to simplify the entire process.

How it Works

Rave or DFM

This is where the ticket coming from. Due to security concern, the tool could not call Rave\DFM API to read case information, instead, we provide an UI for users to manually type in case ID, choose the case type and severity.

Eyes on Calendar

This is a separate tool created by APGC Azure support team, which is widely used by China CSS support team. The tool provides a centralize calendar for team members to mark their on-duty status, like work on duty, annual leave, sick leave, training, morning shift and so on. Our solution will call the API of Eyes on calendar to understand team members’ availability status in real time.

Engineer Profile

The engineer profile is a simple SharePoint list which contains all team members alias, name, specialties, shift, and languages.

Algorithm 

Here’s the basic algorithm for our solution. Basically, all the support engineers are in a queue, and they are ordered by the last time they took case. Engineers who are in the front of the queue are ones who have taken minimum cases in today and yesterday. Once an engineer is assigned a case, the engineer will be moved to the end of the queue. When try to assign a case to an engineer, the Bot will also check engineer’s availability and designed skills to ensure that the engineer is appropriate for the case. In case the first engineer is not entitled for the current new case, the Bot will move to the next engineer for checking till to find a qualified resource for the ticket.
